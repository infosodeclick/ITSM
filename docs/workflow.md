# Workflow Notes

## Asset Assign

1. IT selects an available asset.
2. IT selects the employee.
3. System updates current holder fields.
4. System writes audit log.
5. Future phase creates a signed acceptance record.

## Asset Transfer

1. IT selects an asset currently assigned to an employee.
2. IT chooses the new employee, department, branch, and reason.
3. System records before and after state.
4. System updates current holder.
5. System writes audit log.

## Asset Return

1. HR or IT identifies employee return case.
2. IT checks assigned assets.
3. IT records return condition and destination status.
4. System clears holder or assigns the asset back to stock.
5. System writes audit log.

## HR Onboarding

1. HR submits new employee request.
2. IT receives request.
3. IT prepares device and accounts.
4. IT marks ready to deliver.
5. Delivery assigns assets to the employee.

## HR Offboarding

1. HR submits offboarding request.
2. IT reviews employee-held assets and accounts.
3. IT receives devices and closes access.
4. System records return status and audit log.
