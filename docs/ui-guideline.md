# UI Guideline

แนวทางนี้ใช้สำหรับสร้างหน้าจอ IT Inventory / ITSM ให้เหมาะกับการใช้งานจริงในองค์กร ไม่ใช่หน้า landing page หรือเว็บโฆษณา

## Product Direction

- ระบบเป็น operational tool สำหรับ IT, HR, Manager และผู้ตรวจสอบ
- หน้าจอต้องอ่านง่าย ทำงานเร็ว และลดการกรอกข้อมูลผิด
- ภาษา UI หลักใช้ภาษาไทยที่ทีมงานเข้าใจง่าย
- เริ่มทำทีละ feature หลังจากวางแผนและตรวจผลกระทบแล้ว

## Layout

- หน้าแรกของระบบจริงควรเป็นพื้นที่ทำงาน เช่น dashboard, table, form หรือ task queue
- ถ้ามี dashboard ให้เอาสรุปสำคัญและกราฟไว้ด้านบน แล้วตามด้วย filter และตารางรายละเอียด
- Filter ต้องอยู่ใกล้ข้อมูลที่ filter มีผล เช่น ปีซื้อ, ประเภท, แผนก, สาขา, สถานะ
- ตารางต้องรองรับข้อมูลจำนวนมาก อ่านง่าย และไม่ทำให้ layout กระโดด
- หลีกเลี่ยง card ซ้อน card และ section ที่เป็นกล่องลอยโดยไม่จำเป็น

## Components

- ปุ่มหลัก เช่น เพิ่มข้อมูล, บันทึก, Export ให้อยู่ตำแหน่งที่ผู้ใช้คาดเดาได้
- ใช้ select/dropdown สำหรับข้อมูลที่ควบคุมได้ เช่น สถานะ, ประเภท, สาขา, แผนก
- ใช้ badge แสดงสถานะ เช่น พร้อมใช้, ใช้งานอยู่, พัง, ซ่อม, จำหน่าย, สต็อก
- ฟอร์มต้องมี validation และแสดง error ที่แก้ไขได้ชัดเจน
- Action ที่เปลี่ยนข้อมูลต้องมี feedback ว่าสำเร็จหรือผิดพลาด

## Visual Style

- ใช้สีอย่างมีหน้าที่ ไม่ใช้สีตกแต่งมากเกินไป
- ใช้ความโค้งไม่เกิน 8px สำหรับ card/panel เว้นแต่ component เดิมกำหนดไว้
- ไม่ใช้ hero ขนาดใหญ่ gradient หนัก หรือ decoration ที่ไม่ช่วยงาน
- ตัวอักษรต้องพอดีกับพื้นที่ ไม่ล้นปุ่ม ตาราง หรือ card
- dashboard ควร scan ได้เร็ว แยกตัวเลขสำคัญ กราฟ และรายการแจ้งเตือนชัดเจน

## Reporting UI

- รายงานต้องยึด column contract ที่ผู้ใช้กำหนดในแต่ละ feature
- Export ต้องรักษาลำดับ column และรองรับภาษาไทยใน Excel
- ตาราง report ควรมี filter ก่อน export เพื่อให้ผู้ใช้ตรวจข้อมูลก่อนดึงไฟล์
